# Academic Personal Homepage

A minimal, folder-based academic homepage for GitHub Pages.  
Built with plain HTML + CSS + a small JS loader. Math rendered via [KaTeX](https://katex.org/).

## File Structure

```
.
├── index.html              ← Main landing page (About + News)
├── styles.css              ← Shared stylesheet
├── common.js               ← JS that auto-loads items from items.json
├── build.py                ← Script to generate items.json from folders
│
├── research/
│   ├── index.html          ← Research listing page (auto-generated)
│   ├── items.json          ← Generated by build.py
│   └── robust-representation/
│       ├── meta.json       ← Metadata for this project
│       ├── index.html      ← Detail page (optional)
│       └── thumbnail.jpg   ← Thumbnail image (optional)
│
├── publications/
│   ├── index.html          ← Publications listing page
│   ├── items.json          ← Generated by build.py
│   ├── example-paper-icml2025/
│   │   ├── meta.json
│   │   └── index.html
│   └── example-paper-jmlr2024/
│       └── meta.json
│
├── teaching/
│   ├── index.html          ← Teaching listing page
│   ├── items.json          ← Generated by build.py
│   └── cs101-spring2025/
│       ├── meta.json
│       └── index.html
│
├── contact/
│   └── index.html          ← Contact page
│
└── .github/workflows/
    └── build.yml           ← Auto-runs build.py on push
```

## How to Add a New Item

### 1. Create a folder

```
publications/my-new-paper/
```

### 2. Add `meta.json`

**Publication:**
```json
{
  "title": "Paper Title",
  "authors": "Author A, <u>Your Name</u>, Author B",
  "venue": "ICML",
  "year": 2025,
  "summary": "One-sentence description.",
  "tags": ["topic"],
  "links": {
    "paper": "https://arxiv.org/abs/...",
    "code": "https://github.com/..."
  }
}
```

**Research project:**
```json
{
  "title": "Project Name",
  "year": 2024,
  "status": "Ongoing",
  "summary": "Brief description.",
  "tags": ["topic A", "topic B"]
}
```

**Teaching course:**
```json
{
  "title": "CS 101 — Intro to CS",
  "semester": "Spring 2025",
  "role": "Instructor",
  "year": 2025,
  "summary": "Course description."
}
```

### 3. (Optional) Add more files

- `thumbnail.jpg` — auto-detected and shown in listing
- `index.html` — detail page (copy from an existing example)

### 4. Rebuild

```bash
python build.py
```

This scans all folders, reads `meta.json`, and regenerates `items.json`.

If you set up the GitHub Action (`.github/workflows/build.yml`), this runs automatically on every push.

## Math Support

KaTeX is loaded on every page. Use:

- Inline: `$E = mc^2$`
- Display: `$$\int_0^\infty e^{-x}\,dx = 1$$`

## Deploy to GitHub Pages

1. Create a repo on GitHub (e.g., `username.github.io` or any name).
2. Push all files to `main`.
3. Go to **Settings → Pages → Source**: select `main`, root `/`.
4. Site will be live at `https://username.github.io/` (or `/repo-name/`).

The GitHub Action will auto-run `build.py` and commit updated `items.json` on each push, so you only need to add folders and push.
